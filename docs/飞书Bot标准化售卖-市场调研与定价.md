# 飞书 Bot 标准化售卖：市场调研与定价（Q1 验证版）

## 1. 结论先行

- 我无法直接抓取咸鱼/小红书的实时页面数据（价格、销量、点赞收藏等），因此不能给出“精确统计”。
- 但可以用一套统一口径在 1-2 天内完成调研，并据此确定你的“引流款/标准款/专业款/托管订阅”定价与标题关键词。

## 2. 你要卖的不是“机器人”，而是“场景闭环”

更容易成交的包装方式：
- 监控告警上飞书（技术团队）
- 表单/线索/工单通知与分派（运营/销售团队）
- 日报周报与定时提醒（管理/项目团队）

## 3. 调研目标（Q1 只回答 3 个问题）

- 竞品主要卖哪 3 种“场景包”？
- 用户更愿意为“代搭建/交付源码/托管运维”中的哪一种付费？
- 你的引流价该设在什么区间，才能拿到足够咨询量？

## 4. 咸鱼调研（偏成交）

### 4.1 搜索词

- 平台词：飞书机器人 / 飞书 Bot / 飞书自动化 / 飞书群机器人
- 技术词：飞书 webhook / 飞书消息推送 / 飞书开放平台
- 场景词：告警通知 / 定时提醒 / 日报周报 / 表单通知 / 工单派发

### 4.2 采集字段（每条都抄，别凭感觉）

- 标题
- 标价（到手价、是否“起”）
- 服务形态（代搭建/交付脚本或源码/托管订阅）
- 交付内容（几个流程、是否含部署、是否含文档）
- 交付周期（当天/1-3天/一周）
- 售后（天数、含不含改需求）
- 热度指标（想要数/浏览/留言/评价/“已售”痕迹）

### 4.3 建议样本量

- 每个关键词抓 10 条，合计 40-60 条就足够做第一轮定价。

## 5. 小红书调研（偏线索）

### 5.1 搜索词

- 平台词：飞书机器人 / 飞书自动化
- 运营词：SOP / 私域 / 自动化提效 / 线索跟进
- 对标词：企业微信自动化 / RPA

### 5.2 采集字段

- 笔记主题（运营向/技术向/模板向）
- 互动指标（赞/藏/评）
- 评论区高频需求（例如：群发、定时、表单、审批、提醒）
- 价格表达（明码标价/私信报价/引流到企微）
- 成交路径（评论关键词→私信→加微→成交）

### 5.3 建议样本量

- 挑互动最高的 30 篇，重点看“评论里用户在问什么”。

## 6. 可执行的定价梯度（先跑通闭环）

### 6.1 引流款（咸鱼主推，拿咨询）

- 价格：¥199-399
- 交付：单场景落地（告警/表单通知/定时提醒三选一）+ 1 次远程协助 + 7 天答疑
- 边界：不含长期托管；超出范围按“新增流程/对接系统”计费

### 6.2 标准款（最容易规模化）

- 价格：¥699-1299
- 交付：2-3 个流程 + 简单配置文档 + 30 天售后

### 6.3 专业款（企业向，做高客单）

- 价格：¥1999-4999
- 交付：多流程编排（含失败重试/去重/频控）+ 日志与告警 + 灰度/回滚方案

### 6.4 托管订阅（把一次性变复购）

- 价格：¥199-599/月 或 ¥1999-5999/年
- 内容：云端部署、告警、版本升级、日常小改（限定工时/次数）

### 6.5 增值项（防止售后失控）

- 新增一个流程：¥299-799/个
- 对接一个新系统：¥999-2999/个
- 私有化部署/内网环境：¥3000 起

## 8. 合规与风险提示

- 不做灰产、不做绕过付费。
- 不提供“账号托管、群控、代运营引流”类高风险服务。
- 客户数据与密钥（如 Webhook/Token）不落盘不外泄，交付前写清边界与责任。
